### Mais Sobre o Conteúdo

Esse post é um relato pessoal de Rafael Bittencourt, um cientista da computação brasileiro especializado em IA, sobre sua jornada com o "vibe coding". O vibe coding é uma abordagem de programação onde você descreve intenções ou ideias em linguagem natural (como se estivesse "vibing" ou transmitindo uma vibe), e a IA (como modelos de linguagem grandes, ou LLMs) transforma isso em código executável, muitas vezes sem que o usuário entenda completamente o código gerado. É uma forma de desenvolvimento impulsionado por IA, onde o foco está em prompts conversacionais e colaboração com ferramentas de IA, em vez de codar manualmente do zero. Não é preguiça, mas uma maneira eficiente de prototipar e construir apps rapidamente, especialmente para quem não é expert em todas as áreas (como frontend para um data scientist).[ibm.com](https://www.ibm.com/think/topics/vibe-coding)+5 mais

No post, Rafael destaca sua evolução: começou em 2024 com ferramentas como GitHub Copilot, Windsurf e Cursor, enfrentando frustrações como código ruim e frontends feios. Em 2025, ele "explodiu" com ferramentas CLI agentic como Codex CLI (provavelmente uma CLI baseada em GPT-5 ou similar da OpenAI) e Claude Code (da Anthropic), investindo em múltiplas contas premium para maximizar resultados. Ele enfatiza a importância de mindset persistente, estruturas de controle (como prompts e arquivos personalizados), e uso de ferramentas top de linha, mesmo que custem caro. Como exemplo, menciona o projeto DocIA, uma solução de IA para medicina, onde ele foca em estratégia e delega execução para agents.

Isso reflete uma tendência maior em 2025-2026: o vibe coding está se tornando acessível para não-programadores ou especialistas parciais, permitindo criar apps full-stack (backend + frontend) com bancos de dados e autenticação. Rafael também critica quem usa ferramentas "modestas", argumentando que vale investir em premium para qualidade e velocidade. Em outro post dele, ele recomenda superar ferramentas como Replit ou Firebase Studio e ir para Claude Code e Codex CLI para mais controle e complexidade.[cloudflare.com](https://www.cloudflare.com/learning/ai/ai-vibe-coding/)+2 mais

### Mais Dicas sobre Vibe Coding

Baseado no post original e em buscas por conteúdos semelhantes no X e na web, aqui vão dicas adicionais para melhorar no vibe coding. Compilei de fontes variadas para dar uma visão ampla:

1.  **Mantenha repositórios privados no GitHub**: Evite expor código gerado por IA publicamente para proteger ideias e evitar problemas de segurança.[@pokeapallascat](https://x.com/pokeapallascat/status/2005857504799121431?referrer=grok-com)
2.  **Use o modo de planejamento (Plan Mode)**: No Claude Code, pressione Shift + Tab duas vezes para entrar no modo de planejamento antes de executar. Isso ajuda a estruturar projetos complexos e evita erros. Cycle entre brainstorming, planning e execution para construir confiança com o agent.[@pokeapallascat](https://x.com/pokeapallascat/status/2005857504799121431?referrer=grok-com)[@omarsar0](https://x.com/omarsar0/status/1984641893519839271?referrer=grok-com)
3.  **Configure sub-agents e agents personalizados**: Crie agents específicos como "session summaries agent", "review critic" ou "security audit agent" no Claude Code. Você pode até chamar Codex CLI como um agent dentro do workflow para inputs independentes. Para workflows criativos, combine Claude Code com Bash e Skills para tarefas como edição e formatação.[@pokeapallascat](https://x.com/pokeapallascat/status/2005857504799121431?referrer=grok-com)+2 mais
4.  **Interrompa o agent quando necessário**: Não tenha medo de pausar o Claude Code se algo parecer errado – isso mantém o controle.[@pokeapallascat](https://x.com/pokeapallascat/status/2005857504799121431?referrer=grok-com)
5.  **Planeje a estrutura do projeto com antecedência**: Dedique tempo extra para mapear arquitetura, milestones e deliverables testáveis. Use prompts context-rich e conversacionais para tratar a IA como colaboradora, não só ferramenta.[@pokeapallascat](https://x.com/pokeapallascat/status/2005857504799121431?referrer=grok-com)[@mikegiannulis](https://x.com/mikegiannulis/status/2005632425934594385?referrer=grok-com)
6.  **Monitore o uso de contexto**: No Claude Code, use comandos como /context para checar limites, /cost para custos, /model para trocar modelos, e /resume para continuar chats. Rode em "Skip Permissions Mode" (Yolo Mode) para agilizar.[@rileybrown](https://x.com/rileybrown/status/1966275792427712978?referrer=grok-com)
7.  **Invista em ferramentas premium**: Como Rafael, use múltiplas contas Ultra/Pro de Claude Code, Codex CLI e Gemini CLI. Evite modelos modestos – o custo se paga em produtividade. Atualize direto para planos Max.[@pokeapallascat](https://x.com/pokeapallascat/status/2005857504799121431?referrer=grok-com)
8.  **Integre APIs e bancos de dados**: Para apps reais, use InstantDB para autenticação e bancos, ou APIs como Nano Banana para features como editores de thumbnails. Para mobile, construa e deploye via plataformas como VibeCodeApp.[@rileybrown](https://x.com/rileybrown/status/1966275792427712978?referrer=grok-com)
9.  **Aprenda com guias e vídeos**: Há um guia completo de vibe coding com Claude Code e Codex, construindo 12 apps (incluindo landing pages, lead magnets e apps mobile). Inclui timestamps para prompts iniciais, edições e deploy. Outro vídeo foca em sub-agents para maximizar utilidade.[@rileybrown](https://x.com/rileybrown/status/1966275792427712978?referrer=grok-com)[@Hesamation](https://x.com/Hesamation/status/1973128664188387519?referrer=grok-com)
10.  **Comece simples e itere**: Para iniciantes, foque em static pages ou marketing vibes (como landing pages com Tally para leads). Avalie resultados com múltiplos agents e refine. Lembre: persistência é chave, como Rafael passou de frustrações em 2024 para sucessos em 2025.[@rileybrown](https://x.com/rileybrown/status/1966275792427712978?referrer=grok-com)

Essas dicas vêm de experiências reais compartilhadas por vibe coders. Se quiser aprofundar em uma ferramenta específica (como Claude Code), posso buscar mais detalhes!
